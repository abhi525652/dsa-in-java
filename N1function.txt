# 🎯 What is a Function?

A **Function** is a reusable block of code that performs a specific task.

Instead of writing the same code multiple times, we write it once and call it whenever needed.

---

# 📝 Function Syntax

```java
returnType functionName(parameters)
{
    // code
}
```

### Example

```java
static void greet()
{
    System.out.println("Good Morning");
}
```

---

# ▶️ Function Call

A function **never runs automatically**.

We must call it.

```java
greet();
```

### Execution Flow

```
main()

   │
   ▼

greet()

   │
Print "Good Morning"

   │
   ▼

Back to main()
```

---

# 📥 Parameters vs Arguments

## Parameter

A variable declared inside the function.

```java
static void show(int x)
```

Here,

```
x → Parameter
```

---

## Argument

The actual value passed while calling the function.

```java
show(10);
```

Here,

```
10 → Argument
```

---

# 🏠 Local Variable

A variable declared inside a function.

```java
static void demo()
{
    int x = 5;
}
```

After the function finishes,

```
x is destroyed.
```

It cannot be accessed outside the function.

---

# 🔄 Why do we use `return`?

The **return** keyword sends a value back to the function caller.

### Example

```java
static int add(int a, int b)
{
    return a + b;
}
```

Calling

```java
int answer = add(5,3);
```

Execution

```
add(5,3)

↓

return 8

↓

answer = 8
```

---

# ❌ Without `return`

```java
static void addOne(int n)
{
    n = n + 1;
}
```

```java
int x = 5;

addOne(x);

System.out.println(x);
```

Execution

```
x = 5

↓

n = 5

↓

n = 6

↓

Function Ends

↓

Output = 5
```

### Why?

Only **n** changed.

The original variable **x** remains unchanged.

---

# ✅ With `return`

```java
static int addOne(int n)
{
    n = n + 1;

    return n;
}
```

```java
int x = 5;

x = addOne(x);

System.out.println(x);
```

Execution

```
x = 5

↓

n = 5

↓

n = 6

↓

return 6

↓

x = 6
```

Output

```
6
```

---

# 🎯 Very Important Difference

## Wrong

```java
addOne(x);
```

Returned value is ignored.

---

## Correct

```java
x = addOne(x);
```

Returned value is stored.

---

# ⚔️ `void` vs `return`

| `void` | `return` |
|----------|-----------|
| Returns Nothing | Returns a Value |
| Performs an Action | Sends Result Back |
| No Value to Store | Value Can Be Stored |
| Example: Print Message | Example: Add Numbers |

---

# 🔢 Primitive Variables

```java
static void change(int x)
{
    x = 100;
}
```

```java
int a = 10;

change(a);

System.out.println(a);
```

Output

```
10
```

## Memory

```
a = 10

↓

x = 10

↓

x = 100

↓

x Destroyed

↓

a = 10
```

✅ Java passes a **copy** of primitive values.

---

# 📦 Arrays in Functions

```java
static void change(int arr[])
{
    arr[0] = 100;
}
```

```java
int arr[] = {10,20,30};

change(arr);

System.out.println(arr[0]);
```

Output

```
100
```

## Memory

```
Main arr
      │
      ▼
+----+----+----+
|10  |20  |30  |
+----+----+----+
      ▲
      │
Function arr
```

Both references point to the **same array**.

---

# 📌 Primitive vs Array

| Primitive | Array |
|------------|--------|
| Copy is passed | Reference to same array is passed |
| Original doesn't change | Original can change |

---

# 🔥 Golden Rules

- ✅ A function runs only when called.
- ✅ Parameters receive arguments.
- ✅ Local variables are destroyed after function execution.
- ✅ `return` sends a value back.
- ✅ Ignoring the returned value means losing it.
- ✅ `void` returns nothing.
- ✅ Primitive variables are copied.
- ✅ Arrays can be modified inside functions.

---

# 💼 Interview Questions

## 1. Why do we use `return`?

**Answer**

We use the `return` keyword to send a value from a function back to the caller.

---

## 2. When should we use `void`?

**Answer**

Use `void` when the function performs an action but does not need to return any value.

---

## 3. What is the difference between Parameter and Argument?

| Parameter | Argument |
|------------|----------|
| Variable in Function | Actual Value Passed |

---

## 4. Why doesn't changing an `int` inside a function change the original variable?

Because Java passes a **copy** of primitive values.

---

## 5. Why does changing an array inside a function affect the original array?

Because both references point to the **same array object**.

---

# 🧠 Quick Revision

```
Need a Result?
        │
   Yes ─┴──► Use return
        │
        ▼
Store the returned value.

-----------------------------

Need only an Action?
        │
   Yes ─┴──► Use void
```

❌ **No → Use `void`**
